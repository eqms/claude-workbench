<objective>
Replace the predictable temp-file path in `pdf_export.rs` with `tempfile::Builder`, eliminating
the symlink-redirect attack vector (CR-03 / SEC-04).

Purpose: On shared Linux/XRDP systems a local attacker can pre-create a symlink at the
predictable path `/tmp/<project>-<stem>-<date>.html` and redirect the preview write to an
arbitrary file (e.g. `~/.ssh/authorized_keys`). `tempfile::Builder` opens with `O_EXCL`,
making the attack impossible.

Output: `default_preview_file()` returns `Result<tempfile::NamedTempFile>`; callers store the
handle in `App::temp_preview_files: Vec<tempfile::NamedTempFile>` so the file survives long
enough for the browser to read it, then is auto-deleted on drop.
</objective>

From 01-RESEARCH.md — target API (VERIFIED against tempfile-3.24.0):
```rust
use tempfile::Builder;

pub fn default_preview_file(
    source: &Path,
    project_name: &str,
) -> std::io::Result<tempfile::NamedTempFile> {
    let stem = source.file_stem().and_then(|s| s.to_str()).unwrap_or("preview");
    let prefix = if project_name.is_empty() {
        format!("{}-", stem)
    } else {
        format!("{}-{}-", project_name, stem)
    };
    Builder::new()
        .prefix(&prefix)
        .suffix(".html")
        .tempfile_in(std::env::temp_dir())
}
```

Drop timing pitfall (from 01-RESEARCH.md Pitfall 3):
- `NamedTempFile` is deleted when it drops. Store it in `App::temp_preview_files` or the
  browser will open a missing file.
- Pattern: push the handle BEFORE calling the browser open function.
