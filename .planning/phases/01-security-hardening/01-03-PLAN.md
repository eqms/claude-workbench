<objective>
Add an allow-list guard (`validate_program`) to `opener.rs` that blocks shell metacharacters in
browser/editor config before `Command::new` executes (WR-01 / SEC-02). Replace the hand-rolled
`split_command` with `shlex::split` for correct single-quote handling. Remove the `$SHELL -i -c`
interactive-shell fallback from `dependency_checker.rs` (WR-02 / SEC-03).

Purpose: Both functions process user-provided config strings and pass them to OS command
execution. An injection surface exists if config is ever derived from untrusted input. The
`split_command` UX bug (`open -a 'Brave Browser'` misparsing) is fixed as a byproduct.
The shell fallback in the dependency checker is an injection pattern that is unnecessary —
direct `Command::new` finds all currently-probed binaries on a normal PATH.

Output: `validate_program` rejects metacharacters; `shlex::split` replaces `split_command` in
`opener.rs`; `$SHELL -i -c` block removed from `dependency_checker.rs`.
</objective>

```rust
fn validate_program(prog: &str) -> anyhow::Result<()> {
    if prog.is_empty() || !prog.chars().all(|c| {
        c.is_ascii_alphanumeric() || matches!(c, '_' | '-' | '.' | '/' | '+')
    }) {
        anyhow::bail!("Unsafe program name in browser/editor config: {:?}", prog);
    }
    Ok(())
}
```
