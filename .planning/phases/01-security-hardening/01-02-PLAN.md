<objective>
Gate `--update-to` behind `#[cfg(debug_assertions)]` so release binaries cannot be silently
downgraded to vulnerable versions (CR-02). Also backport IN-02: filter one-shot flags in
`restart_application()` to eliminate the infinite downgrade loop.

Purpose: Any local user can currently run `claude-workbench --update-to 0.1.0` in release builds,
bypassing any future signature verification by downgrading to an unsigned release. Gating the flag
to debug builds removes this surface entirely. The IN-02 backport eliminates the complementary
infinite-restart bug at zero additional cost — both fixes live in the same two files.

Output: Release build: `--update-to` is unknown to clap (exits 2). Debug build: flag works as
before. `restart_application()` strips `--update-to`, `--check-update`, `--clipboard-diag`,
`--ssh-paste-diag` from the re-exec args.
</objective>
